WHAT HAPPENS WHEN YOU HOST A JAM

HOST · STEP 1
CODE GENERATION

Host clicks "Start Jam". CloudBeat calls genJamCode() — Math.random() → base36 → toUpperCase → 6 chars. e.g. A3K7PZ. A unique UID is also generated for the host.

HOST · STEP 2
FIREBASE ROOM CREATED

CloudBeat calls Firebase set(ref(db, 'rooms/A3K7PZ'), {...}). Writes: host info, repo source config, current track state (name, position, paused, timestamp). Room is now live.

HOST · STEP 3
CODE SHOWN IN JAM BAR

The Jam Bar appears below the top navigation showing A3K7PZ with a pulsing live indicator. Host shares this code verbally or via copy button. The code is visible until the session ends.

LISTENER · STEP 4
ENTERING THE JAM CODE

Listener opens CloudBeat → clicks Jam icon → enters the 6-char code. CloudBeat validates the code against Firebase. If the room exists, it reads the source config (GitHub repo) and loads it.

LISTENER · STEP 5
REPO SYNC + REALTIME

Listener fetches the same GitHub repo as host. Then subscribes via onValue() to the Firebase room. Every state update from host is received instantly — track changes, seeks, play/pause.

SYNC · STEP 6
CONTINUOUS SYNC LOOP

Host writes state to Firebase every 2 seconds via jamWrite(). On track change or seek, it writes immediately. Listeners receive these and auto-seek to match the host's position with lag compensation.

OPTIONAL · STEP 7
GRANT LISTENER CONTROL

Host can click a listener's name chip in the Jam Bar and grant them DJ control. That listener can now change tracks and seek. Host can revoke at any time. Only one listener can have control at once.

END · STEP 8
LEAVING THE JAM

When Host leaves, Firebase writes hostGone: true and the room is removed. All listeners are notified and automatically disconnected. Listener presence entries in rooms/{code}/listeners are cleaned up.

Complete Setup Guide

HOW TO INSTALL & USE CLOUDBEAT

Follow these steps once to get your own music player with Jam, Google Drive, and shared listening — fully working and hosted for free.

💡
IMPORTANT — JAM SESSION AUDIO NOTE

To listen to audio together in a Jam, your audio files must be hosted on GitHub — not Google Drive. Google Drive files are private to the file owner and cannot be accessed by other listeners. GitHub repos with audio files act as a shared public CDN that everyone in the Jam can fetch from simultaneously.

01
Firebase · Required for Jam Sessions
CREATE A FIREBASE PROJECT & GET YOUR API KEY

Firebase powers the real-time Jam session sync. You need a free Firebase Realtime Database for the Jam feature to work.

aGo to console.firebase.google.com and sign in with Google.
bClick Add project → give it a name (e.g. my-cloudbeat) → click through the setup wizard.
cIn the left sidebar go to Build → Realtime Database → click Create Database → choose any region → start in test mode.
dGo to Project Settings (gear icon top-left) → scroll to Your apps → click </> Web → register your app.
eFirebase shows you a firebaseConfig code block. Copy the entire snippet — you'll paste it in the Generate panel below.
📋 The config looks like: const firebaseConfig = { apiKey: "AIza...", authDomain: "...", databaseURL: "https://...", ... }
02
Google Cloud · Required for Google Drive & Sign-In
CREATE A GOOGLE OAUTH CLIENT ID

This lets users sign in with Google and access their Drive. You'll set this up in Google Cloud Console (it's free).

aGo to console.cloud.google.com → select the same Google account.
bCreate a new project or select your existing Firebase project from the dropdown.
cGo to APIs & Services → Library → search Google Drive API → click it → click Enable.
dGo to APIs & Services → Credentials → click Create Credentials → OAuth 2.0 Client ID.
eChoose Web application as the type. Under Authorized JavaScript origins add your final hosted domain (e.g. https://yoursite.netlify.app). You can add it later too.
fClick Create. Copy the Client ID — it looks like 123456-abc.apps.googleusercontent.com.
⚠️ If Google Drive sign-in doesn't work after deploy, come back here and add your Netlify/Vercel domain to the authorized origins list.
03
Generate · Your personalised CloudBeat file
GENERATE & DOWNLOAD YOUR INDEX.HTML

Use the Generate Your CloudBeat button in the section above. Paste your Firebase config snippet and your OAuth Client ID — the generator injects your credentials into the CloudBeat app and downloads a ready-to-deploy index.html. Everything happens locally in your browser.

🔒 Your API keys never leave your browser. The generator modifies the file client-side and triggers a direct download — no server involved.
04
GitHub · For hosting your player file
CREATE A GITHUB REPO & UPLOAD YOUR PLAYER

GitHub stores your CloudBeat player file (and optionally your music files). You need a GitHub account — it's free.

aGo to github.com/new → create a new public repository (e.g. cloudbeat-player).
bUpload your downloaded index.html to this repo. You can drag-and-drop it in the GitHub web UI.
cOptional — for Jam audio: Create a separate repo (e.g. my-music) and upload your MP3/FLAC/WAV files there. This is the repo you'll enter in CloudBeat's settings.
05
Deploy · Host your player on the web for free
DEPLOY ON NETLIFY OR VERCEL

Both Netlify and Vercel host static sites for free. Connect your GitHub repo and deploy in minutes.

aGo to netlify.com or vercel.com → sign in with GitHub.
bClick Add new site → Import an existing project → select your cloudbeat-player GitHub repo.
cLeave all build settings blank (it's a plain HTML file) → click Deploy.
dYour site will be live in ~30 seconds at a URL like https://cloudbeat-player.netlify.app.
06
Domain · Your CloudBeat address
COPY YOUR LIVE DOMAIN & SHARE IT

Copy the URL Netlify or Vercel gave you. This is your CloudBeat — share it with anyone you want to listen with. They don't need to install anything, just open the link.

📎 Remember to also add this domain to your Google Cloud OAuth authorized origins (Step 2e) so Google sign-in works properly.
07
First Use · Start listening
OPEN THE APP, LOGIN & ADD YOUR MUSIC

Open your CloudBeat URL in any modern browser.

aLogin: Sign in with Google (for Drive access & sync) or enter any name as a Guest.
bAdd GitHub music: Go to Settings → enter your music repo URL in the format owner/repo-name (e.g. yourname/my-music) → CloudBeat scans and loads all audio files.
cAdd Google Drive music (personal only): Sign in with Google → browse your Drive → play any audio or video file directly.
08
Jam Sessions · Listen together in real-time
HOST A JAM & INVITE FRIENDS

Once your music is loaded from GitHub, you can start a Jam and listen together with anyone who has the link.

aYou (Host): Click the Jam button in CloudBeat → click Host a Jam → a 6-character code appears (e.g. A3K7PZ).
bShare the code: Send your friends the website URL + the Jam code. They open the same URL.
cFriends join: They sign in as Guest or with Google → click Jam Session → enter the code → they're instantly synced to your playback.
dEveryone hears the same track in real-time — play, pause, skip, and seek are synced across all listeners.
🎵 Jam only works with GitHub-hosted audio. Each listener fetches the audio file directly from GitHub — so the files must be in a public repo.

🔒 Privacy

CloudBeat collects zero data. There are no analytics, no tracking, and no telemetry of any kind.

What CloudBeat stores: Your settings, playlists, liked songs, EQ, and theme are stored locally in your browser's localStorage (guest mode) or in a single JSON file in your own Google Drive (Google mode). This data never leaves your device unless you explicitly sync it yourself.

What CloudBeat does NOT collect: No email addresses, no listening history sent to the developer, no device fingerprinting, no cookies, no analytics SDKs.

Firebase Jam Sessions: When you host or join a Jam, the room code, your chosen display name, and playback state are written to Firebase. This data is temporary — it is deleted when the session ends. It is readable by anyone who knows the room code.

Google OAuth: When you sign in with Google, your name, profile photo, and email are used to display your identity in the UI. They are stored in memory only — not in localStorage or any server.

⚠️ Do not share your Jam room code publicly if you want to keep your listening session private. Anyone with the code can join and see what's playing.
