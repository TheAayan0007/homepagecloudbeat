WHAT HAPPENS WHEN YOU HOST A JAM

HOST · STEP 1
CODE GENERATION

Host clicks "Start Jam". CloudBeat calls genJamCode() — Math.random() → base36 → toUpperCase → 6 chars. e.g. A3K7PZ. A unique UID is also generated for the host.

HOST · STEP 2
FIREBASE ROOM CREATED

CloudBeat calls Firebase set(ref(db, 'rooms/A3K7PZ'), {...}). Writes: host info, repo source config, current track state (name, position, paused, timestamp). Room is now live.

HOST · STEP 3
CODE SHOWN IN JAM BAR

The Jam Bar appears below the top navigation showing A3K7PZ with a pulsing live indicator. Host shares this code verbally or via copy button. The code is visible until the session ends.

LISTENER · STEP 4
ENTERING THE JAM CODE

Listener opens CloudBeat → clicks Jam icon → enters the 6-char code. CloudBeat validates the code against Firebase. If the room exists, it reads the source config (GitHub repo) and loads it.

LISTENER · STEP 5
REPO SYNC + REALTIME

Listener fetches the same GitHub repo as host. Then subscribes via onValue() to the Firebase room. Every state update from host is received instantly — track changes, seeks, play/pause.

SYNC · STEP 6
CONTINUOUS SYNC LOOP

Host writes state to Firebase every 2 seconds via jamWrite(). On track change or seek, it writes immediately. Listeners receive these and auto-seek to match the host's position with lag compensation.

OPTIONAL · STEP 7
GRANT LISTENER CONTROL

Host can click a listener's name chip in the Jam Bar and grant them DJ control. That listener can now change tracks and seek. Host can revoke at any time. Only one listener can have control at once.

END · STEP 8
LEAVING THE JAM

When Host leaves, Firebase writes hostGone: true and the room is removed. All listeners are notified and automatically disconnected. Listener presence entries in rooms/{code}/listeners are cleaned up.

Quick Start

START IN 60 SECONDS

01
OPEN THE FILE

Download index.html and open it in Chrome, Firefox, Safari, or Edge. No server needed — it runs locally or from any static host.

02
LOGIN OR GUEST

Sign in with Google for full Drive sync, or enter any name for an instant guest session. Your data is stored locally until you decide to link Google.

03
ADD YOUR MUSIC

Create a playlist from a GitHub repo (enter owner/repo) or browse your Google Drive for audio files. CloudBeat scans everything automatically.

04
START A JAM

Hit the Jam button, get your code, share with friends. Everyone hears the same music in sync — no matter where in the world they are.

🔒 Privacy

CloudBeat collects zero data. There are no analytics, no tracking, and no telemetry of any kind.

What CloudBeat stores: Your settings, playlists, liked songs, EQ, and theme are stored locally in your browser's localStorage (guest mode) or in a single JSON file in your own Google Drive (Google mode). This data never leaves your device unless you explicitly sync it yourself.

What CloudBeat does NOT collect: No email addresses, no listening history sent to the developer, no device fingerprinting, no cookies, no analytics SDKs.

Firebase Jam Sessions: When you host or join a Jam, the room code, your chosen display name, and playback state are written to Firebase. This data is temporary — it is deleted when the session ends. It is readable by anyone who knows the room code.

Google OAuth: When you sign in with Google, your name, profile photo, and email are used to display your identity in the UI. They are stored in memory only — not in localStorage or any server.

⚠️ Do not share your Jam room code publicly if you want to keep your listening session private. Anyone with the code can join and see what's playing.
